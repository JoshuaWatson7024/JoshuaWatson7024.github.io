What I learned from this article about git

  • The "repository" is actually inside the hidden .git folder
  • If you just start you cmd line and enter "git init" you'll put your whole hard drive under git control!
  • When you commit a file, you give it to git. At that point, you edit and commit copies that git gives you
  • A commit takes a "snapshot" of the ENTIRE working tree (every file you've committed ever to this repository)
  • "Adding" or "staging" files sends them to the "index" or "stage"
  • git status shows you "what's new"
  • Every commit (except the root commit) has a parent commit, this allows history tracking
  • One commit can have multiple children - this is how you make branches
  • "git switch" will let you switch branches
  • when you "git merge" git creates a new commit with two parents - the last commit of the working tree and the specified tree
  • "remote" holds the name of the URL of an online copy of the repository
  • "origin" holds the url of a cloned repository
  • "git fetch" updates your remote tracking branches (local on your machine)
  • If the remote repo has branches you don't have, you need to fetch and merge before pushing
